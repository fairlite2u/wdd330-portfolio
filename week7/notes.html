MDN Web Docs: Cross-Origin Resource Sharing(CORS)

  • Cross Origin Resource Sharing or CORS is an HTTP-header based mechanism that allows a server to indicate any other origins than its own from which a browser should permit loading of resources.
  • Simple requests must meet the following conditions:
    • One of the allowed methods: GET, HEAD, or POST.
    • Only these headers are allowed to be manually set: Accept, Accept-Language, Content-Language, and Content-Type.
    • The only allowed values for the Content-Type header are: application/x-www-form-urlencoded, multipart/form-data, and text/plain.
    • If the request is made using an XMLHttpRequest object, no event listeners are registered on the object returned by the XMLHttpRequest.upload property used in the request.
    • No ReadableStream object is used in the request.
  • For Preflighted requests, the browser first sends an HTTP request using the OPTIONS method to the resource on the other origin, in order to determine if the actual request is safe to send.
  • Requests with credentials require a specific flag to be set on the XMLHttpRequest object or the Request constructor.

JavaScript: Novice to Ninja Chapter 11: Further Functions

Further Functions

  • A callback is a function that's provided as an argument to another function.
  • Promises can be used instead of callback to deal with multiple asynchronous actions in sequence.
  • A generator function (created by placing an * after the function keyword) will return an iterator object that provides a next() method, which returns the next value in a sequence that is defined in the generator function.
  • Functional programming involves breaking processes down into steps that can be applied as a series of functions.
  • Higher-order functions are functions that accept another function as an argument or return another function as a result, or both.
  • Currying is a process that involves the partial application of functions and relies on higher-order functions that are able to return partially applied functions.

JavaScript: Novice to Ninja Chapter 13: Ajax

Ajax

  • Ajax is a technique that allows web pages to communicate asynchronously with a server and dynamically updates web pages without reloading.
  • Allows for partial page updates.
  • Ajax can be used to communicate with external APIs and requests can be made using the Fetch API.
  • The Response interface deals with the object returned when the promise is fulfilled.
  • The data is most often sent in JSON, but can be sent in other forms.
  • The Request object allows for more fine-grained control over the request being made.
  • The Headers interface can be used to create a Headers object, which can be added as a property of Request and Response.
  • The FormData interface helps to reduce the amount of code needed when submitting forms.
