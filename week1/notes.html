Week 1 Notes


Doing Stuff With Web Things

  • Think, Design, Test, Create - not the "Oh Crap!" method.
  • HTML - define the stuff to be shown to the user.
  • CSS - make the stuff shown pleasant to see.
  • JavaScript - make the user experience interactive rather than passive.
  • Make things describable as a function for practice, i.e. takeShower or makeDinner
  • Figure out what, if anything, is going in, what, if anything, is coming out, and what may be happening as part of what is being done.
  • Always give a function a name that is a verb.
  • If you are dealing with an input or textarea use value; a section, use innerHTML.
  • Think about key-value pairs & describe them, i.e. in gradebooks, the student name is the key, the grades are the values.
  • Make sure you use == to compare if two things are equal and = to put something in a variable.
  • != means "not equal to"; ! means NOT; && means AND.
  • The parseInt function tries to convert any text to an integer.
  • NaN means "not a number".
  • isNan function can be used to determine if input is "not a number".
  • The standard for loop parts when working with an array are:
    1. The for keyword
    2. A variable to keep track of the location you are at in the array
    3. A check to see if the loop should continue
    4. An update that increases the location tracking variable by one
    5. The lines of code to execute everytime you go around the loop
