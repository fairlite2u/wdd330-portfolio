HTML5 & CSS3 For the Real World Chapter 8: CSS3 Transforms and Transitions

Transforms

  • The CSS3 transform property lets you translate, rotate, scale, and/or skew any element on the page.
  • Manipulate an element's appearance using transform functions:
    • transform: translate(x,y); transform: translateX(); transform: translateY() - allows you to move elements left, right, up, or down.
    • transform: scale(x,y); transform: scaleX(); transform: scaleY() - Scales and element by the defined factors horizontally and vertically.
    • transform: rotate() - Rotates an element around the point of origin by a specified angle value.
    • transform: skew(x,y); transform: skewX(); transform: skewY() - Specifies a skew along the x and y axes.
    • transform:matrix(n,n,n,n,n,n) - Combines all the 2D transform methods into one.
  • There is also a transform-origin property that controls the origin from which the transforms are applied.

Transitions

  • The CSS3 transition property allow the values of CSS properties to change over time, essentially providing simple animations.
  • Steps to create a simple transition using only CSS:
    1. Declare the orignial state of the element in the default style declaration.
    2. Declare the final state of your transitioned element; for example, a :hover state.
    3. Include the transition functions in your default style declaration using the transition properties.
  • Make sure to note that the transition is declared in the default or originating state.
  • Transition properties:
    • transition-property - Defines the CSS properties of the element that should be transitioned, with all for all properties being the default.
    • transition-duration - Sets how long the transition will take, specified in seconds or milliseconds.
    • transition-timing-function - Controls the pace of the transition in more granular detail.
    • transition-delay - Allows you to delay the start of the transition.
    • The transition shorthand property allows you to combine the above 4 values into one declaration.
  • The transition property can be applied to the transform property.

Animations

  • CSS animations allow you to control each step of an animation via keyframes.
  • A keyframe is a snapshot that defines a starting or end point of any smooth transition.
  • Animations allow us to add any number of keyframes inbetween the start and end points.
  • To create an animation, use the @keyframes rule followed by a name of your choosing.
  • Animation properties:
    • animation-name - This property is used to attach an animation to an element.
    • animation-duration - This property defines the length of time an animation takes to complete one iteration.
    • animation-timing-function - This property determines how the animation will progress over its duration.
    • animation-iteration-count - This property lets you define how many times the animation will play through.
    • animation-direction - This property allows you to control what direction and animation plays in.
    • animation-delay - This property is used to define how many milliseconds or seconds to wait before the browser begins the animation.
    • animation-fill-mode - This property defines what happens before the first animation iteration begins and after the last animation iteration concludes.
    • animation-play-state - This property defines whether the animation is running or paused.
    • The shorthand animation property allows you to declare values for all or some of the properties at once.

HTML5 & CSS3 For the Real World Chapter 12: Canvas, SVG, and Drag and Drop

Canvas

  • HTML5's Canvas API allows us to draw anything through JavaScript.
  • Canvas can be used for animation, game graphics, data visualization, photo manipulation, and real-time video processing.
  • If the height and width are different than the default, it's best to explicitly specify the attributes in the canvas element

SVG

  • SVG stands for Scalable Vector Graphics, which is a specific file format that allows you to describe vector graphics using XML.
  • Unlike canvas, images created with SVG are available via the DOM.
  • Image editors, like Inkscape (inkscape.org) can be used to help create SVG images.

Drag and Drop

  • The Drag and Drop API allows us to specify that certain elements are draggable, and then specify what should happen when these draggable elements are dragged over or dropped onto other elements on the page.
  • Not supported by Android or iOS.
  • There are two major kinds of functionality that can be implements with Drag and Drop: dragging files from your computer into a web page (in combination with the File API) or dragging elements into other elements on the same page.
  • Steps to adding drag and drop to your page:
    1. Set the draggable attribute on any HTML elements you'd like to be draggable.
    2. Add an event listener for the dragstart event on any draggable HTML element.
    3. Add an event listener for the dragover and drop events on any elements that you want to have accept dropped items.
  • draggable is not a Boolean attribute, so you have to explicitly set it to true.
  • DataTransfer objects allow us to set and get data about the elements that are being dragged.
