Week 3 Notes


Object methods, "this"

"this" in methods

  • It's common that an object method needs to access the info stored in the object, for example, the code inside user.sayHi() may need the name of the user. That's when the this keyword comes into play.
  • The value of this is the object "before dot", the one used to call the method.

"this" is not bound

  • In JavaScript, this can be used in any function, even if it's not a method of an object.
  • The value of this is evaluated during run-time, depending on the context.
  • The rule is simple: if object.method() is called, then this is object during the call of method.

Arrow functions have no "this"

  • If we reference this from an arrow function, it is taken from the outer "normal" function.
  • Provides the advantage of being able to use both types of quote marks within a string: `She said, "It's Me!"`

This in JavaScript

So, what is this?

  • this is a keyword whose value changes depending on how a functions gets called.
  • Preferred method of creating an array is using the array literal notation: const myArray = [];
  • There are six different ways where this can take on new values:
    1. this in global context: not usually used
    2. this in object construction: this refers to the object instance
    3. this in an object method: this refers to the object, use methods to get the instance of an object
    4. this in a simple function: this is always set to Window
    5. this in an arrow function: this is always the same as this around it (in it's immediate scope).
    6. this in an event listener: this is set to the element that fired the event.

Changing this with bind

  • bind is a method that is present in every function.
  • bind allows you to change the this context.
  • This method takes in any number of arguments and returns the bound function.
  • The first parameter you pass into bind becomes this in the bound function.
  • The other parameters you pass will be passed as arguments to the original function.
  • Note: bind doesn't work with arrow functions.

Object Literals

  • A self contained set of related values & functions.
  • If a property's value is a function, it is known as a method.
  • Think of it as like a dictionary where you look up a property name and see a value.
  • Each property is a key-value pair.

Built-in Objects

  • JSON - a string representation of the object literal notation.
    • Differences:
      1. Property names must be double-quoted.
      2. Permitted values are double-quoted strings, numbers, true, false, null, arrays, and objects.
      3. Functions are not permitted values.
  • Math Object - all the properties and methods of the math object are immutable and unable to be changed.
  • Date Object - contains information about dates & times.
  • The RegExp Object

The Document Object Model

  • DOM, for short, represents an HTML document as a network of connected nodes that form a tree-like structure.
  • The DOM treats everything on a web page as a node.
  • The HTML tag is the root node and every other part is a child node.
  • The DOM is not actually part of the JavaScript because it is language agnostic. This means it can be used in any programming language.

Getting Elements

  • The DOM provides several methods that allow us to access any element on a page:
    • getElementById()
    • getElementsByTagName()
    • getElementsByClassName()
    • Query Selectors
