JavaScript: Novice to Ninja Chapter 2: Programming Basics

Variables

  • ES6 onward uses const and let over var
  • const is used when the variable will not be reassigned to another value
  • let is used if the variable might be reassigned
  • const should be used to declare most variables
  • Using const does not prevent non-primitive data types from being mutated

Scope

  • Using const and let means they are block scoped (or local scope), so their value only exists inside the block they are declared in
  • A variable declared outside a block has global scope, which is not good practice as they are accessible anywhere in program. Keep those to a minimum

Template Literals

  • Use the backtick character (`) to deliminate strings
  • Provides the advantage of being able to use both types of quote marks within a string: `She said, "It's Me!"`
  • Also allows interpolation which means that a JavaScript expression can be inserted inside a string and the result will be displayed

JavaScript: Novice to Ninja Chapter 3: Arrays, Logic, and Loops

Arrays

  • An ordered list of values
  • Preferred method of creating an array is using the array literal notation: const myArray = [];
  • Be careful when using the delete operator to delete items from an array because it replaces the value with 'undefined'
  • Understand what methods are non-destructive or destructive to arrays and how they can be permanently changed

Sets

  • A data structure that represents a collection of unique values
  • Currently no literal notation - create with the new operator: const list = new Set();
  • If set already contains a value and you try to add that same value, it will be ignored

Maps

  • A convenient way of keeping a list of key and value pairs
  • Currently no literal notation - create with the new operator: const romanNumerals = new Map();

JavaScript: Novice to Ninja Chapter 4: Functions

Functions

  • You can define a function a few different ways - function declarations, function expressions, Function() constructor (not recommended)
  • Be consisten with whatever method you use
  • Arrow functions are much more succinct but are best for short, one line, functions

Callbacks

  • A function that is passed as an argument to another function
  • Helps with separating functions out so that the same code isn't repeated in multiple functions
