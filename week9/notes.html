JavaScript: Novice to Ninja Chapter 9: The Window Object

The Browser Object Model

  • The Browser Object Model is a collection of properties and methods that contain information about the browser and computer screen.
  • These properties and moethos are made available through the window object.
  • Global variables are created without using the const, let, or var keywords and are actual properties of the global object which means they are also properties of the window object.
  • window object functions that produce dialogs: alert(), confirm(), prompt(). These methods should be user with care becuase they stop the executions of a program in its tracks.

Browser Information

  • The window object has a number of properties and methods that provide information about the user's browser including navigator and location. These also have their own properties that provide more information, including:
    • window.navigator.userAgent - Returns information about the brower and operating system being used. This property has been deprecated from the official specification, but remains well supported in all major browsers.
    • window.location.href - Returns the full URL as a string.
    • window.location.protocol - Returns a string describing the protocol used.
    • window.location.host - Returns a string describing the domain of the current URL and the port number (which is often omitted if the default port 80 is used).
    • window.location.pathname - Returns a string of the path that follows the domain.
    • window.location.assign() - Can be used to load another resource from a URL provided as a parameter.
  • Most of those properties are read/write, which means they can also be changed by assignment.
  • The window.history property can be used to access information about any previously visited pages in the current browser session, not to be confused with the HTML5 History API.
  • The window.screen object contains information about the screen the browser is displayed on. Things like height, width, available height and width, and color bit depth can be found. The screen object has more uses for mobile devices.
  • Many of these methods and properties should be used with care as they have been abused in the past.

The Document Object

  • The document object has properties and methods that deal with the page that has been loaded into the window. Here are a few methods worth looking at:
    • document.write - Writes a string of text to the page. If a page has already loaded, it will completely replace the current document. You can also include HTML in the string and it will become part of the DOM tree. Placing the method within a document inside <script> tags will inject the string into the markup and not overwrite the rest of the HTML on the page. The use of document.write is heavily frowned upon and there are only rare legitimate uses of it.
    • document.cookie - Used to create a cookie which is a list of name/value pairs. Cookies are used to allow a browser to retrieve information between requests to remember user's previous visits. Using cookes for data storage is being replaced by the HTML5 localStorage API in many cases, but cookies are still useful for retaining state information such as if a user is logged in.
    • document.write

Timing Functions

  • The window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second paramenter.
  • window.clearTimeout() will prevent the timeout if called before the setTimeout parameters are met.
  • The window.setInterval() method will repeatedly invoke the callback function after every given number of milliseconds.
  • window.clearInterval() will stop the setInterval method.
  • setTimeout() and setInterval() can be used to animate elements on a web page, although not as well as window.requestAnimationFrame().
  • window.requestAnimationFrame() has a much smoother performance.
  • In general, it is typically better to use CSS for any animation effects, although there may be times when JavaScript might be the better solution.

JavaScript: Novice to Ninja Chapter 14: HTML5 APIs

The data- Attribute

  • A way of embedding data in a web page using custom attributes that are ignored by the browser.
  • They are private to a page, so are not intended to be used by an external service - their sole purpose is to be used by a JavaScript program.
  • When naming these attributes, they must use the following format:
    • Start with date-.
    • Contain only lowercase letters, numbers, hyphens, dots, colons, or underscores.
    • Include an optional string value.

HTML5 Web Storage

  • The Web Storage API provides a key-value store on the client's computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use.
  • Browsers that support the Web Storage API have a window object property called localStorage that has several properties and methods that are used to store data.
  • There is also a sessionStorage object that works like localStorage but the data is only saved for the current session.
  • Basics of using localStorage:
    • To set and item and then retrieve it's value, you can use localStorage.setItem('name', 'value') and localStorage.getItem('name').
    • Alternatively, you can use assignment, for example: localStorage.name = 'value'.
    • To remove something from local storage you can use the removeItem method or the delete operator or you can remove everything using the clear method.

Geolocation

  • The Geolocation API is used to obtain the geographical position of the device.
  • Permission to use this has to be granted by the user first, due to privacy concerns.
  • Geolocation is a property of the navigator object.

Web Workers

  • Since JavaScript is a single-threaded language, web workers allow processes to be run in the background, adding support for concurrency in JavaScript.
  • The Worker() constructor function is used to create a new worker: const worker = new Worker('task.js');.
  • The function takes another JavaScript file as an argument and downloads it. Once the file has downloaded completely, the worker will start.
  • The variable that is assigned to the constructor function is used to refer to the worker in the main program, the keyword self is used to refer to the worker in the worker script.
  • Web worker use the concept of messages to communicate back and forth between the main script and the worker script.
  • worker.terminate() is used to stop a worker from inside the main code or self.close() in used from inside the worker script.
  • Shared web workers can also be created to allow lots of different scripts on the same domain to access the same worker object.
  • The Service Worker API allows a worker script to run in the background with the added benefit of being able to intercept network requests.

Websockets

  • Websocket is a protocol that allows two-way communication with a server - also know as push messaging.
  • Check out websocket.org for more ways to use websockets and to learn more about them.

Notifications

  • The Notification API allows you to show messages using the system's notifications.
  • Permission must be granted by the user to send notifications, which can be achieved by using the requestPermission() method.

Multimedia

  • HTML5 introduced the audio and video tags to insert audio and video clips into a web page, along with a Media API for controlling the clips.
  • The audio and video elements bring native support for multimedia into the browser and the API gives developers full control of the playback of audio and video.
