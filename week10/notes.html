JavaScript: Novice to Ninja Chapter 9: The Window Object

The Browser Object Model

  • The Browser Object Model is a collection of properties and methods that contain information about the browser and computer screen.
  • These properties and moethos are made available through the window object.
  • Global variables are created without using the const, let, or var keywords and are actual properties of the global object which means they are also properties of the window object.
  • window object functions that produce dialogs: alert(), confirm(), prompt(). These methods should be user with care becuase they stop the executions of a program in its tracks.

Timing Functions

  • The window.setTimeout() method accepts a callback to a function as its first parameter and a number of milliseconds as its second paramenter.
  • window.clearTimeout() will prevent the timeout if called before the setTimeout parameters are met.
  • The window.setInterval() method will repeatedly invoke the callback function after every given number of milliseconds.
  • window.clearInterval() will stop the setInterval method.
  • setTimeout() and setInterval() can be used to animate elements on a web page, although not as well as window.requestAnimationFrame().
  • window.requestAnimationFrame() has a much smoother performance.
  • In general, it is typically better to use CSS for any animation effects, although there may be times when JavaScript might be the better solution.

JavaScript: Novice to Ninja Chapter 14: HTML5 APIs

The data- Attribute

  • A way of embedding data in a web page using custom attributes that are ignored by the browser.
  • They are private to a page, so are not intended to be used by an external service - their sole purpose is to be used by a JavaScript program.
  • When naming these attributes, they must use the following format:
    • Start with date-.
    • Contain only lowercase letters, numbers, hyphens, dots, colons, or underscores.
    • Include an optional string value.

HTML5 Web Storage

  • The Web Storage API provides a key-value store on the client's computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use.
  • Browsers that support the Web Storage API have a window object property called localStorage that has several properties and methods that are used to store data.
  • There is also a sessionStorage object that works like localStorage but the data is only saved for the current session.
  • Basics of using localStorage:
    • To set and item and then retrieve it's value, you can use localStorage.setItem('name', 'value') and localStorage.getItem('name').
    • Alternatively, you can use assignment, for example: localStorage.name = 'value'.
    • To remove something from local storage you can use the removeItem method or the delete operator or you can remove everything using the clear method.
